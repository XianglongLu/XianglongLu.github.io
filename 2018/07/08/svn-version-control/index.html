SVN —— 基本命令的使用

直接切入正题，介绍SVN的一些常用指令。

1. export(ep)，checkout(co)和commit(ci)：

# export最新版本到本地
$ svn ep svn://xianglonglu/xxx/yyy

# checkout最新版本到本地
$ svn co svn://xianglonglu/xxx/yyy

# export第5个版本到本地
$ svn ep -r 5 svn://xianglonglu/xxx/yyy

# 将修改的内容commit到仓库，日志信息写在冒号里面。
svn ci -m "author: lxl, date:20180708, action: for test."

# 将修改的内容commit到仓库，日志信息写在文件log.txt里。
svn ci -F ../log.txt

2. 批量添加文件

当你将一些新的文件添加到checkout的版本里并输入

$ svn st

之后，你会发现你添加的文件前面会被列出来，前面的符号为”?”，
你修改的文件前面是”M”。
如果你添加的文件较少，使用

$ svn add xxx.c

便可把它添加好。再输入svn st会发现它前面变成了”A”。
但是如果文件很多的时候，你可以通过如下方法来实现：

svn st | awk '{if($1=="?"){print $2}}' | xargs svn add

你在输入svn st，你会发现你新添加的文件前面会变成”A”。

3. 查看内容

# 查看第5个版本的版本日志
svn log -r 5

# 查看第5个版本的中改动的文件
svn log -r 5 -v

# 查看当前仓库的信息
svn info

# 查看第5个和本地版本的差别
svn diff -r 5

# 查看第5个和第6个版本的差别
svn diff -r 5:6

# 查看第5个版本中test.c和本地test.c的差别
svn diff -r 5 test.c

# 查看第5个和第6个版本中test.c的差别
svn diff -r 5:6 test.c

4. SVN仓库的回滚

假设你当前仓库的有0-7个版本，你想删除第6和第7版本。
具体操作如下：
来到你的仓库目录，假设为repo_xxx

# 新建一个repo叫new_repo
svnadmin create new_repo
# 将原来的repo的第0到第5版本dump到new_repo中
svnadmin dump -r 0:5 repo_xxx | svnadmin load new_repo
# 将配置配件拷贝过来
cp -rf repo_xxx/conf/* new_repo/conf/
# 修改权限
chmod o+rwx -R new_repo
# 将旧仓库更名或者删除（随你了，其实最好可以更名留作备份）
mv repo_xxx old_repo
# 将新仓库更名
mv new_repo repo_xxx

至此，完成了6，7版本的删除。其实操作很灵活，你可以试着只删除中间的某个版本，方法类似。

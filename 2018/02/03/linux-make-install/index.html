Linux —— 如何实现软件的多版本共存？

Linux下们常常需要通过软件/工具包的源代码（source code）来完成安装工作。其中会涉及到3个指令：

$ ./configure
$ make
$ make install

以前不假思索，按顺序敲完这三个指令，便完成了软件的安装。

如今为了实现一些软件的多版本共存问题，需要将软件安装到特定目录时，便不知所措了。

通过阅读一些文章，本文介绍如下2种将软件安装到指定路径的方法。

方法一：在./configure时配置安装路径

$ ./configure --prefix=/PATH
$ make
$ make install

其中PATH为你要将软件安装到的目录。

方法二：在make install时指定安装路径

$ ./configure
$ make
$ make install DESTDIR=/PATH

PATH为你要将软件安装到的目录

实例分析 — 实现cmake 2.8和3.5的版本共存

当前Linux系统默认cmake版本为2.8，我们要做的是将3.5版本安装到非默认路径在软连接到/usr/bin下。

首先下载cmake-3.5.1.tar.gz，解压得到：cmake-3.5.1，并按照方法1，在configure的时候就将其安装路径配置到非默认安装路径下（我这里选择安装到/usr/local/test目录下）：

$ cd cmake-3.5.1
$ ./configure --prefix=/usr/local/test
$ make
$ make install

在/usr/bin/下创建软连接，并命名为cmake-3.5.1：

$ ln –s /usr/local/test/bin/cmake /usr/bin/cmake-3.5.1

测试是否成功：

$ cmake-3.5.1 –version

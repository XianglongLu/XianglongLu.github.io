前言

自己对嵌入式系统的学习有一段时间了，对很多知识也都有了一些粗浅的了解。
其中一个知识点 —— embedded Linux的启动过程，对于初学者来说可谓是一大难点。
今天我就根据自己的了解，简单的描述一下其中的过程。
如有不正确的地方，欢迎大家批评指正。

1. 基础概念简介

1.1 ROM / 只读内存

ROM（Read Only Memory）：只读内存，即常说的存储空间，掉电后数据不会丢失。相当于电脑中存储数据的硬盘。

ROM包含有多种类型：
1 Flash（闪存），又分为NAND Flash和NOR Flash。
NAND Flash属于顺序存取，NOR Flash属于随机存取。
人们常说的eMMC（embedded multi media card）则是将Flash和控制器封装到了一起。
现在较好的手机上则使用的是UFS(universal flash storage)。
2 PROM（Programmable ROM，可编程只读存储器）
3 EPROM（Erasable Programmable，可擦可编程只读存储器）
4 EEPROM（Electrically Erasable Programmable，电可擦可编程只读存储器）

1.2 RAM / 随机存储(存取)内存

RAM（Random-Access Memory）全称随机存取内存，即我们常说的运存（运行内存）。
它的特点在于掉电后里面的数据会全部消失。
所谓“随机存取”，指的是当存储器中的数据被读取或写入时，所需要的时间与这段信息所在的位置或所写入的位置无关。
相对的，读取或写入顺序访问(Sequential Access)存储设备中的信息时，其所需要的时间与位置就会有关系(如磁带)。

1 SDRAM(Synchronous Dynamic RAM)
2 DDR(double data rate RAM)，主要用于笔记本和台式机上。
3 LPDDR(Low Power double data rate RAM)，主要用于移动设备上。

1.3 NVRAM / 非易失性随机访问内存

NVRAM（Non-Volatile Random Access Memory）
断电后数据不会丢失。

1.4 MMU / 内存管理单元

2. Linux系统的软件架构概述

一个嵌入式Linux系统从软件的角度可以分为四个部分：
1 BootLoader 即引导程序，常用BootLoader有u-boot，cfe（broadcom），类似于PC上的BIOS
2 Kernel 内核
3 FileSystem 文件系统
4 Application 应用程序

3. Linux启动过程介绍

3.1 BootLoader阶段

我们把image烧录到嵌入式产品中，实际上它是被烧录到产品的Flash（ROM）中。

当系统上电/复位后，系统最先运行的是第一段代码就是BootLoader。而该段代码的地址一般位于一个特定的地方。

BootLoader一般会包含2种模式：启动加载模式和下载模式。（1）启动加载模式：启动加载模式的整个过程不需要操作者的介入，它自动从ROM中将操作系统加载到RAM中来。（2）下载模式：该种模式主要是供开发者使用。一般在BootLoader默认进入启动加载模式前，往往有几秒钟的等待时间（面向客户的系统往往会把该时间设得很短甚至为0秒），开发者可在此时通过按下特定的按键让系统进入下载模式（u-boot和cfe一般是按下回车键）。进入该模式后通过一定的选项设置，则可将存储在别的地方的Kernel镜像文件或文件系统文件（例如存储在与网线相连的PC上或U盘等外置存储设备中）下载到RAM中来，再由RAM将这些文件写到ROM中。

BootLoader会完成处理器和外部设备的初始化工作，然后再启动Kernel（注：如果你使用的是nand flash，它最后做的一件事就是 —— 把Kernel拷贝到RAM中，并跳转到RAM中的Kernel处开始执行；而如果你使用的是nor flash，它会直接在跳转到Kernel处开始运行。）

3.2 Kernel阶段

ZTEXTADDR: 解压
ZRELADDR:
TEXTADDR: kernel启动的虚拟地址，TEXTADDR = PAGE_OFFSET + TEXTOFFSET（kernel偏移地址）

Kernel的启动过程大体上可以分为2个阶段：Kernel镜像的自引导和Linux Kernel子模块的初始化。

Kernel镜像的自引导主要完成2件事情（1）完成Kernel镜像的解压decompress_kernel()；（2）进入Kernel代码的入口call_kernel。然后执行start_kernel()函数。

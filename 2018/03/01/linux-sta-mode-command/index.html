Linux —— 通过命令将Linux设备连接到WIFI热点(wpa_supplicant和dhcpcd)

为什么想到要这么做？

对于一些无屏幕或无图形界面的Linux设备，通过命令行来完成对WIFI热点的连接可能成为了我们的唯一方法。

工具介绍和准备

所需准备的两个工具：

wpa_supplicant
dhcpcd

wpa_supplicant

wpa_supplicant是wifi client加密认证工具，它支持wep、wpa、wpa2等完整的加密认证。
注：如果你要建立一个ap端，你需要的加密认证工具是hostapd。
wpa_supplicant启动后会再后台运行，此时我们可以通过wpa_cli来进行操作控制。

dhcpcd

dhcpcd，即dhcp client daemon。用于从dhcpd获取IP地址。

搜索附近可用的WIFI热点：

$ iw dev wlan0 scan             // 会罗列出所有信息
$ iw dev wlan0 scan | grep SSID    // 筛选出SSID信息

假设我们搜索到了一个WIFI热点的名称（SSID）叫test，密码为12345678，加密方式为WPA或WPA2。
那么接下来需要怎么连接它呢？

Step 1：配置ssid和password

在/etc/wpa_supplicant.conf（该文件具体位置依据系统而定）下增加：

network={
ssid="Test"
psk="12345678"
priority=1
}

若还想添加其他WIFI热点，按照该模板再接着添加即可：

network={
ssid="ssid/索要连接的热点名称(注意大小写)"
psk="密码"
}

Step 2：后台运行wpa_supplicant

$ wpa_supplicant -D nl80211 -i wlan0 -c /etc/wpa_supplicant.conf &
// &表示后台运行
// -D driver name
// -c configuration file
// -i interface name

Step 3：启动dhcpcd获取IP地址

$ dhcpcd wlan0 &

Step 4：检查是否连接成功

4.1 查看是否连接上

iw wlan0 info

查看wlan0 interface的信息，看看ssid是否是你之前想要连接的那个。

4.2 是否获得IP地址

ifconfig

查看wlan0 interface下的inet addr行是否获得了IP地址等信息。

至此，你应该已经连上了WIFI热点。

将Hexo从Github部署到Coding.net

前言 / Preface

心血来潮，利用GitHub + Hexo搭建了个人的博客。但访问速度一直比较慢。偶然之间得知GitCafe的存在（已被Coding.net收购）。尝试把博客部署到Coding.net上。


正文 / Body

首先，你需要申请一个Coding.net的账号。这里就不再赘述了。直接进入正题。

1. 在Coding.net上建立一个新项目（Create your Coding.net repository）

登入Coding.net后，点击右上角”+”号，然后选择“项目”。

然后填写项目的“基础选项”。
项目名称我写的和我的用户名一样，项目描述也随便填一下。
至于项目的私有或公开，随你喜欢。这里我选择私有。
最后点击“创建项目”。至此，项目建立完毕。

2. 配置你Hexo博客源码下的_config.yml文件

打开你Hexo博客源码下的_config.yml文件。找到部署（deploy）的位置，并添加你Coding.net的仓库信息，如第5行所示。

deploy:
  type: git
  repo: 
        github: https://github.com/XianglongLu/XianglongLu.github.io.git,master
        coding: https://git.coding.net/XianglongLu/XianglongLu.git,master

3. 给Coding.net添加SSH Key

想必你在使用GitHub时已经生成了SSH Key，路径：C:\Users\Administrator（这里是你的用户名）.ssh\id_rsa.pub。
打开id_rsa.pub文件，将里面的内容复制到“SSH-RSA公钥内容”里面。“公钥名称”随意填写。

完成SSH Key的添加后，我们可以在Git Bash中通过指令来验证添加是否成功。
在Git Bash中输入如下指令：

ssh -T git@git.coding.net

得到类似于如下的回复则表示SSH Key添加成功了。

Coding 提示: Hello XianglongLu, You've connected to Coding.net via SSH. This is a personal key.
XianglongLu，你好，你已经通过 SSH 协议认证 Coding.net 服务，这是一个个人公钥

由于我们之后使用Pages服务方式，在这里我们需要在本地Hexo博客的source目录下创建一个名为Staticfile的空文件，该文件是静态文件部署的标志。
你可以手动创建，也可以在Git Bash下依次输入如下命令来创建：

cd source
touch Staticfile

4. 将你本地的Hexo博客同步到你Coding.net的仓库

在Git Bash上输入：

hexo g -d

就能把本地Hexo博客同步到我们的Coding.net里。
最后部署Coding.net Pages服务方式。
至此，大功告成，这是我的Coding.net博客：
http://xianglonglu.coding.me/

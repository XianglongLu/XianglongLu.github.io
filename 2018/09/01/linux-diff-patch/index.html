Linux —— 通过diff和patch命令生成和打补丁

1. diff命令简介

首先简单介绍一下diff命令的用法。
假设有两个文件，一个叫a.txt，一个叫b.txt。
a.txt和b.txt的内容分别为：

1    apple        apple
2    boyy        boy
3    cat        cat
4    dog        eye
5    eye        fire
6    fire        girl    
7    girl

输入如下命令后结果如下：

# diff a.txt b.txt
2c2
< boyy
---
> boy
4d3
< dog
7c6
< girl
---
> girl
\ No newline at end of file

“2c2”表示a.txt的第2行和b.txt的第2行有不同。
“4d3”表示a.txt比b.txt多了第4行内容。

如果在后面加上”-y”，则可并排输出两文件，并显示不同的地方。如下所示：

# diff a.txt b.txt -y -W 30
apple           apple
boyy          | boy
cat             cat
dog           <
eye             eye
fire            fire
girl          / girl

“-W 30”指定了两对比文件之间的栏宽。
“|”表示内容有不同。
“<”表示右边文件b.txt比a.txt少了这行。
“>”表示右边文件b.txt比a.txt多了一行。

2. 使用diff命令生成补丁

假设你有2个工程文件，一个是原始工程文件a，另一个是修改过的工程文件b。
则你可以通过diff命令生成一个patch：

# diff -urN a/a.txt b/a.txt > test.patch
--- a/a.txt    2018-09-01 22:29:19.866708400 +0800
+++ b/a.txt    2018-09-01 22:29:20.977491900 +0800
@@ -1,7 +1,6 @@
 apple
-boyy
+boy
 cat
-dog
 eye
 fire
-girl
+girl
\ No newline at end of file

其中：

-u 显示有差异行的上下文，默认是前后各3行。
-r 递归地对比一个目录和它的所有子目录(即整个目录树)。
-N/--new-file 如果没添加该参数，如果文件test.c仅仅出现在某个目录中，会显示Only in xxx/yyy/: test.c。
如果添加了该参数，会将该文件和一个空文件做对比。其实就是在diff的时候考虑到添加文件的问题。

此时该目录下会生成一个patch，名为test.patch。

3. 使用patch命令打补丁

接下来，我们将补丁打给b/a.txt。

# cd b/a
# patch -p1 < ../test.patch
// 有时会出现以下提示
patching file a.txt
Reversed (or previously applied) patch detected!  Assume -R? [n]

输入y和回车继续执行：

patching file a.txt
Reversed (or previously applied) patch detected!  Assume -R? [n] y

此时a.txt已被打上补丁，同时相同路径下生成了一个未打补丁前的原始文件a.txt.orig。

如果在执行一次之前的命令，则会将反打补丁：

$ patch -p1 < ../a.patch
patching file a.txt

将文件恢复到未打补丁之前。

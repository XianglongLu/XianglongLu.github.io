The brief introduction of DDR3 SDRAM

1. DDR DRAM含义解释

DDR DRAM(一般简称DDR)，即Double Data Rate Dynamic Random Access Memory。
dynamic意为动态地，“动态”指的是定期刷新内存数据的意思。
每个存储单元保存一位信息，即逻辑0或1，这些信息是使用电容两段的电压差来表示。
由于电容的物理特性，即使不进行读写，其所存储的电荷量也会慢慢变少
（半导体晶体管中的电荷每1.2us就会消失，数据也就跟着消失，因此为了保存数据，每1us会重新刷新通电一次以保存数据），
所以需要经常刷新内存数据。刷新内存数据的工作一般交由一个额外电路来完成。
这也就是为什么系统断电后数据会全部消失的原因。

2. DDR的数据频率

DDR1数据传输频率 = 核心频率X2
由于在时钟的高电平和低电平时都传输数据，所以数据传输频率=核心频率X2。
DDR2数据传输频率=时钟频率X2=核心频率X4
DDR2开始出现了时钟频率的概念。DDR2将核心频率倍频，
所以时钟频率=核心频率X2，由于在时钟的高电平和低电平时都传输数据，
所以数据传输频率=时钟频率X2。
DDR3时钟频率=时钟频率X2=核心频率X8
DDR3的时钟频率是核心频率的4倍。
DDR1预读取是2位，DDR2是4位，DDR3为8位。

3. DDR3内部结构

DDR3内部类似一个存储矩阵/阵列。我们可以把它想象为一本表格。
表格总共有8页(8 Logic Bank，简称Bank)，每页表格由行(Row)和列(Column)组成。
寻址流程其实很直观，先指定Bank地址，再指定Row地址，最后根据Column地址找到寻址单元。
由此便实现了随机存取（NAND Flash是顺序存取）。

内存为了保证CPU能正常工作，会在一次传输中传输CPU一个传输周期内所需要的数据。
CPU一个传输周期所能接受的数据容量就是CPU数据总线的位宽，单位是bit。

4. DDR的速度等级

Speed Grade速度等级 (DataRate/tCL-tRCD-tRP)
tCL: CAS Latency，列地址选通脉冲延迟时间。
tRCD: RAS-to-CAS Delay，行地址选通脉冲至列地址选通脉冲的延迟时间。
tRP: RAS Precharge Time，行地址选通脉冲的预充电时间。
这3个值越小越好。其中RAS Row Access Strobe行选通脉冲，CAS Column Access Strobe列选通脉冲。
这里的时间是以时钟周期数为单位。

5. 管脚定义分析

DA Data Address
DQ
时钟线：DDR采用的是查分时钟信号。

DQS bi-directional data strobe 双向数据控制引脚

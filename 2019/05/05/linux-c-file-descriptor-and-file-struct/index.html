温故而知新 ——— 文件描述符、文件结构体 和 inode

文件描述符fd（file descriptor）：
它是一个非负整数。
内核利用文件描述符来访问文件。
当每打开一个现存文件或新建一个文件时，内核会返回一个文件描述符。
读写文件也需要使用文件描述符来指定待读写的文件。
其中文件描述符0，1和2一般对应于标准输入STDIN、标准输出STDOUT和标准错误STDERR。

文件结构体（struct file）：
file这个结构体描述进程中打开的一个文件的信息，会与fd文件描述符关联。
文件结构体是内核中的数据结构，它不会出现在用户空间。
file结构体指示一个已经打开的文件。

inode：
内核中用inode结构表示具体的文件。
每个被创建的文件都有一个inode。
inode包含文件访问权限、属主、组、大小、生成时间、访问时间、最后修改时间等信息。
它是Linux 管理文件系统的最基本单位，也是文件系统连接任何子目录、文件的桥梁。

默认情况下，当启动一个Linux进程的时候，会打开3个文件。
它们分别是标准输入STDIN、标准输出STDOUT和标准错误STDERR，
对应的文件描述符分别为0、1和2。

#include <stdio.h>
#include <fcntl.h>
#include <string.h>
#include <unistd.h>

int main() {

        int fd = open("test.txt", O_RDWR|O_CREAT, 0644);
        // 0644 rw-r--r--
        // 文件所有者有rw权限；用户组只有r权限；一般用户只有r权限
        if(fd<0)
        {
            printf("Error.\n");
            return 1;
        }

        printf("fd = %d\n", fd); // 理论上fd=3
        char *str = "test\n";
        int strLen = strlen(str); // 为5

        int retN = write(fd, str, strLen);
        printf("retN = %d\n",retN);

        if(retN<=0)
        // 写入错误返回-1，写入成功返回成功写入的字符数（包括\n）。
        {
            printf("Error in writing.\n");
            close(fd);
            return 1;
        }
        close(fd);

        return 0;
}

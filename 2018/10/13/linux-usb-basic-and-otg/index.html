USB和OTG简介

名词解释

USB Universal Serial Bus 通用串行总线
OTG On-The-Go 网译：数码伴侣/一键拷贝
UDC USB Device Controller
UVC USB Video Class
UAC USB Audio Class
mass storage 大容量存储器
几种USB控制器类型：
HCI Host Controller Interface 主机控制接口
OHCI Open Host Controller Interface 开放式主机控制接口
OHCI支持USB1.1标准。
UHCI Universal Host Controller Interface 通用主机控制接口
UHCI支持USB 1.0和USB 1.1的接口标准，与OHCI不兼容。
EHCI Enhanced Host Controller Interface 增强型主机控制接口
EHCI就是通常所说的USB 2.0接口。
XHCI eXtensible Host Controller Interface 可扩展型主机控制接口
XHCI主要是面向USB 3.0协议，它不仅支持所有的USB 3.0特性，还同时也向下兼容USB 2.0及其以下的设备。XHCI可以理解为EHCI的增强版。

1. USB概述

在USB的世界里，可分Host和Slave两种角色，PC一般是Host，手机等数码设备通常是Slave。
在Linux中，作为USB Slave的设备被称为gadget。

OTG是在USB 2.0出现之后才提出来的，其主要是解决在没有Host的情况下（如PC机），能让两台USB设备实现数据传输的功能。
e.g.支持OTG功能的手机和普通的U盘。

USB各个标准相应的速度信息如下：

USB Version Name Speed Release Time
USB 1.0 Low-Speed 1.5Mbps(192KB/s) 1996年1月
USB 1.1 Full-Speed 12Mbps(1.5MB/s) 1998年9月
USB 2.0 High-Speed 480Mbps(60MB/s) 2000年4月
USB 2.0(OTG 1.0) High-Speed 480Mbps(60MB/s) 2001年12月
USB 3.0 Super-Speed 5Gbps(640MB/s) 2008年11月

2. USB PIN定义

USB接口一般有4条线组成：VCC(5V)，GND，D+，D-
带OTG功能的设备会多1条线：ID pin

PC端USB的D-和D+均接了15k欧姆的下拉电阻。
当Slave插入PC端的USB口后，会将PC端的D-和D+电压拉高。
当ID pin为低电平（与地短接）时，该Device为Host（也叫A设备）；
当为高电平（一般浮空）时，为Slave（也叫B设备）。

OTG有2种设备：两用OTG设备（可当Host和Slave）和外设型OTG设备（只可当Slave）。

当Slave插入Host端后，Host端会提示有设备插入。它是怎么知道的呢？
原理：当Slave插入后，Host会主动向设备发送命令包，要求Slave告诉Host端其设备信息。

多个USB Slave接入USB Host，Host如何区分Slave？
每一个Slave接入时，Host端的USB总线驱动都会给它分配一个编号（地址）。
Host访问哪一个Slave，发出的命令都含有对应的编号。

USB协议
USB设备通过描述符来描述其功能，标准的USB设备有5种USB描述符：
设备描述符，配置描述符，字符串描述符，接口描述符，端点描述符。


USB设备驱动
HID、usb audio class device、usb video class device、mass storage这四种设备可以开发通用的设备驱动，
只需要修改VID和PID即可。

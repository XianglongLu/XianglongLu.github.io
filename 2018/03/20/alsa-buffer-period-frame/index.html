Linux —— (撰写中)ALSA中buffer, period, frame和sample的关系解析

buffer, period, frame和sample

有3个buffer需要了解

  1. 应用程序开开辟buffer，application buffer。
  2. alsa也会在内存中开辟buffer。
  3. 声卡里有一个硬件buffer（HW Buffer）。

录音：数据采集时候先存在硬件buffer，将buffer填满后，产生一个终端，alsa使用DMA将数据弄到alsa buffer里。
播放：application有一个buffer，填上数据后，会调用alsa接口，alsa把buffer将数据复制到其buffer，再把数据交给 hardware buffer。

alsa buffer：
alsa buffer的尺寸可以通alsa库函数来指定，可以非常大。
所以一次传输buffer的数据可能就会导致无法接受的延时。
所以，ALSA把buffer分割成一系列period，以period作为数据传输单元。
一个period由可以由多个frame组成，每一个frame包含某一时刻的的采样值（sample），对于立体声而言，一个frame包含2个channel。
buffer > period > frame >= sample
1 frame = 1 analog sample

alsa的period size直接影响CPU到底做多少工作。在相同情况下，如果period size太小，就会生成多的中断去处理数据。所以如果你不在意低的延时（low latency），那你可能使用较大的period size。这样你就有更多的CPU cycle去做其他事情。

HW buffer
硬件被设置为48000Hz，2 periods，每个周期1024。那么buffer就是2048，硬件在一个buffer的时候就会产生2次中断，alsa会尽量将buffer保持填满，第一个周期的数据播放完成，那么第三周期的数据也会开始进入，而这个时候正是第二个周期的数据播放(典型的ring buffer功能)。

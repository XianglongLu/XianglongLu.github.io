Linux —— DISPLAY环境变量

曾经在使用PUTTY通过UART调试一款带LCM的embedded device时碰到过这样一个问题。
在PUTTY上输入firefox，显示屏上却没看到firefox browser有打开。

该device是Xorg的架构，我当前通过PUTTY打开的UART console相当于remote login，相当于一个X Client（后称为Client 1）。
而LCM上显示的桌面系统本身又是另一个X Client（Client 2）。
在Client 1上去打开Application，让其显示在Client 2上，显然得需要设置或配置些什么。

要完成这个目标。你需要了解DISPLAY环境变量了。
在Linux上，DISPLAY环境变量用来设置将图形显示到什么地方。

首先在Client 1端输入，查看其DISPLAY环境变量的信息：

# echo $DISPLAY
:0.0

DISPLAY环境变量格式如下host:NumA.NumB
host指X Server所在的主机主机名或者IP地址，图形将显示在这设备上。
NumA为连接的端口减去6000的值，如果NumA为0，则表示连接到6000端口。
NumB的话一般都为0。

其次，想要让其他Client能在该当前Client上显示图形之类的，就必须开放权限：

# xhost +
access control disable, clients can connect from any host

回到Client 2，查看Client 2的DISPLAY环境变量：

# echo $DISPLAY

理论上是空的。
接着设置和Client 1一样的DISPLAY环境变量：

# export DISPLAY=:0.0
:0.0
# echo $DISPLAY
:0.0

至此，设置完毕。
测试一下吧。在Client 2端输出firefox，应该可以看到Client 1端的firefox browser打开了~

Linux —— Device Tree Source/DTS/设备树 简介

设备树是一套用来描述硬件属相的规则。

在出现设备树之前，设备信息都会写入驱动程序中，一旦外围设备发生变化，驱动程序就要进行修改甚至重写。引入设备树的目的就是为了解决这个问题，即将驱动代码与设备信息分离开来。
至此，驱动程序只负责处理驱动的逻辑，设备的相关信息则存储在设备树文件中。在移植或开发中，如果只是设备的信息，如硬件接口信息的变化，而驱动逻辑没有变化，我们则只需要修改设备树文件，不必像以前一样去修改驱动程序。

旧的Linux机制都是用kernel/arch/arm/plat-xxx目录和kernel/arch/arm/mach-xxx目录下用的代码来描述硬件。

对于arm Linux，设备树一般在内核（kernel）”的arch/arm/boot/dts/“目录内。
比如amlogic参考板的板级设备树位于”kernel/aml-4.9/arch/arm64/boot/dts/amlogic/“目录下。

设备树中几个简称的含义：
DTS Device Tree Source 设备树源文件，是文本格式，方便人们阅读。
DTC device tree compiler 设备树编译器，可将DTS转换为DTB。
DTB device tree blob 是DTS的二进制形式，供机器使用的。
DTSI Device Tree Source Include 类似于C语言中的xxx.h文件。

设备树结构解析

设备树是树形结构，且是一棵树。
除了根节点外，每个节点（node）都有唯一的父节点。
节点下可以有子节点和属性。

/{                  // 根节点
    node1{          // node1是节点名，node1是/的子节点
        key=value;  // node1的属性
        ...
        node2{      //node2是node1的节点
            key=value;
            ...
        }
    }               // node1至此描述完毕
    node3{
        key=value;
        ...
    }
}

“/“表示根节点，也可说就是我们的板子。

其他节点的格式为node-name@unit-address，@之前为节点名字，后面为节点地址。

属性（property）
属性是用来描述节点特性的。根据需要可以定义0个到多个属性来描述节点的特性。

属性的组成
属性由名字和值组成。

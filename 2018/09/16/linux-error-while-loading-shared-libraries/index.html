Linux —— 运行可执行文件报错 error while loading shared libraries

error while loading shared libraries: xx.so.x

出现该错误一般有2种可能：

  1. 当前系统中确实没有这个库的存在或者版本不正确；
  2. 有该库的存在，但它却不在系统的默认搜索路径下；

当出现第一种问题时，去安装该库就可以了。
如果你不知道该库来自于什么安装包，可通过如下命令进行搜索：

apt-file search libxxx.so.y

当你把共享库安装到/usr/lib或/lib目录下后，记得执行以下命令

ldconfig

ldconfig命令主要是在默认搜寻库目录(/lib和/usr/lib)以及动态库配置文件/etc/ld.so.conf内所列的目录下，
搜索出可共享的动态链接库搜索出可共享的动态链接库(格式如lib.so)，进而创建出动态装入程序(ld.so)所需的连接和缓存文件。
缓存文件默认为/etc/ld缓存文件默认为/etc/ld.so.cache，此文件保存已排好序的动态链接库名字列表。

如果是第二种情况，可有几种解决方案：
（1）你可以将它们软连接到/lib或者/usr/lib目录下，再执行ldconfig命令。

（2）export一个全局变量LD_LIBRARY_PATH，然后运行程序的时候就会去这个目录中找共享库。

export LD_LIBRARY_PATH=/usr/local/xxx/lib:$LD_LIBRARY_PATH

即，将/usr/local/xxx/lib增加到$LD_LIBRARY_PATH之中。
该方法一般来讲只是一种临时的解决方案，主要用于如果共享库文件安装到了其它”非/lib或/usr/lib”目录下，
但是又不想在/etc/ld.so.conf中加路径(或者是没有权限加路径)。

（3）把共享库目录添加到共享库配置文件/etc/ld.so.conf中。
如下所示，将/usr/local/lib追加到配置文件中：

# cat /etc/ld.so.conf
include ld.so.conf.d/*.conf
# echo "/usr/local/lib" >> /etc/ld.so.conf
# ldconfig

/lib            内核级别的共享库文件
一般被/bin和/sbin中的程序所调用。

/usr/lib        系统级别的共享库文件

/usr/local/lib  用户级别的共享库文件

这些库目录下的库文件命名格式一般有3种：

libxxx.so        

libxxx.so.y      //libxxx.so.y多为libxxx.so的一个软链接/符号链接(symbolic link)

ldxxxx.so

例如输入ls -l后：

-rwxr-xr-x    1 root     root       1259016 Sep  3  2018 libc-2.23.so
lrwxrwxrwx    1 root     root            12 Aug  3  2018 libc.so.6 -> libc-2.23.so

可以看出libc.so.6为libc-2.23.so的一个软链接。

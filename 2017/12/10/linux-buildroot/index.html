Linux —— Buildroot学习笔记

前言 / Preface

Buildroot是一个能够用来搭建Embedded Linux System（嵌入式Linux系统）的交叉编译环境的工具。
通过这个交叉编译环境，我们可以编译出一个完整的、可以直接烧写到板子上运行的嵌入式Linux系统软件。
这其中包含uboot、kernel、rootfs以及rootfs中的各种库和应用程序。
类似的架构还有yocto，有兴趣的小伙伴可以自行搜索。

看了Buildroot user manual和网上的一些文章，写下此文作为读书笔记。

小插曲

记得当年刚入职，老板我：电脑上运行的Linux系统跟嵌入式产品上运行的Linux系统有什么不同？
自己愣是答不到点子上。
几经周转之后才知道：电脑（台式机&笔记本）的CPU一般是x86架构，处理的是CISC指令；
而嵌入式产品的CPU一般是arm或MIPS架构，处理的是RISC指令。
交叉编译环境便是将代码编译为可在嵌入式系统上处理的RISC指令软件/系统包。

正文 / Body

1. Buildroot的目录结构简介

下面简单介绍Buildroot的目录结构。

1.1 Buildroot/docs/

顾名思义，里面存放的是相关的文档。

1.2 Buildroot/output/

这是编译后生成的输出文件夹。
build/下放着的是编译后生成的软件包。
host/下是制作好的编译工具链。
target/下是用于制作根文件系统（rootfs）的相关文件，并不是你自己烧到板子的rootfs。
images/下为可以烧入板子的各种image。

1.3 Buildroot/fs/

里面放着的是文件系统的源代码。

1.4 Buildroot/linux/

linux kernel的自动构建脚本。

1.5 Buildroot/arch/

里面存放着如ARM，MIPS等CPU架构相关的配置脚本。

1.6 Buildroot/configs/

存放嵌入式板子的一些配置参数。

2. u-boot, kernel和rootfs&app的配置文件路径及配置方法

2.1 u-boot

u-boot’s config file dir:

u-boot/configs/

2.2 kernel

kernel’s config file dir:

kernel/arch/arm64/configs/ //假设你的soc是arm64

how to config:
(1)

cd buildroot
make menuconfig

(2)

vim kernel/arch/arm64/configs/xx_defconfig  // 名字不一定是以defconfig结尾

2.3 rootfs & app

rootfs & app’s config file dir:

buildroot/configs/

how to config:
(1)

cd buildroot
make menuconfig

(2)

vim buildroot/configs/xx_defconfig // 名字不一定是以defconfig结尾

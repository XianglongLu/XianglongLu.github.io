Linux —— X-window vs Wayland

简单来说，X Window和Wayland都是显示服务器（Display Server）。
其中，X Window在过去的几十年都占据了很大的份额，而Wayland则是近十几年兴起的一种号称会取代X Window的新显示服务器。

X Window主要由X Client和 X Server组成，且属于Client/Server模式(C/S)。
它们沟通遵循的协议（Protocal）叫X，现在用的一般是X11，即X的第11版。
有了X协议，还需要具体的软件来实现这个协议，这个软件叫做X Server。
实现X协议的X Server有Xfree86、XORG、MOTIF等等。

Wayland则由Wayland Client和Wayland Compositor组成，属于Compositor/Client模式(C/C)。
它们沟通的协议也叫做Wayland，是一个合成协议（compositing protocal）。
对于Wayland协议，按照官网的说法，Weston是Wayland协议的实现。
其实可以简单的理解为Weston中有多个component/function，实现Wayland协议是其中一个部分。

KDE和GNOME(GNU Network Object Environment)是窗口管理器(WM, Window Manager)，
同时提供了桌面、菜单等等，启动它们后我们进入了可视化操作。
为了方便开发人员开发X Client，X协议被封装成了Xlib，然而Xlib仍对开发者不够友好，于是乎它被再次封装Qt。
QT是直接使用Xlib库，GTK通过GDK/GLIB来间接使用Xlib库。

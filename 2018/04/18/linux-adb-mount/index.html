Linux设备与PC进行文件交互之ADB、USB device和tftp

我有一台嵌入式Linux设备，我想将让我的PC（windows/Linux都行）能够与之进行文件交互。
通常有如下几种方法：

  1. ADB
  2. mount USB device
  3. tftp server/client
    下面我就逐一介绍一下。

1. ADB Shell

之前乍一听ADB Shell，我本以为只有安卓设备才可以使用该功能。其实不然。
只要嵌入式设备有USB接口，且设备上装有adbd，你则可通过USB轻松的完成交互。
在你的PC端（假设为windows）安装好adb shell driver，那么你则可通过：
假设将PC端的xxx文件传输到嵌入式设备/media/目录下：

$ adb push xxx /media/

将/usr/bin/下的yyy文件拷贝到PC端当前目录下：

$ adb pull /usr/bin/yyy ./

2. mount USB device

把你要传输的内容存到U盘里，将U盘插入嵌入式设备。
再终端上查看是否检测到USB设备：

$ cat /proc/partitions

假设新增的USB device为sda1，将它挂载某一个地方。
假设我要挂载mnt下面的test目录里：

$ mkdir /mnt/test
$ mount /dev/sda1 /mnt/test

则可把你的USB device mount到/mnt/test目录下。
使用完成后，记得umount：

$ umount /dev/sda1

3. tftp

整体框架如下：
PC（tftp server，ip 192.168.1.100） <--> Linux Device（tftp client，ip 192.168.1.1）
在PC端通过终端（telnet或者串口）控制Linux Device。

在终端中输入如下指令，即可完成相应操作： 

// 从tftp server 192.168.1.100下载xxx文件到Linux Device的当前位置：
tftp -g -r xxx 192.168.1.100

// 将Linux Device当前目录下的xxx文件上传到tftp server 192.168.1.100
tftp -p -l xxx 192.168.1.100

// -l 表示本地文件名， -r 表示远程主机文件名， -g 表示下载文件， -p 表示上传文件
